# R0 Baseline

**Date:** 2026-06-28
**Environment:** local development workstation
**Control plane:** Next.js local dev server
**Execution plane:** local Hermes gateway
**Text model:** `qwen/qwen3.5-9b` through LM Studio
**Image provider:** Kie.ai

These measurements provide a comparison point for later roadmap phases. They
are small functional samples, not load-test or production SLO results.

## Live Workflow Samples

| Workflow | Result | First visible state | Completion | Work | Cost |
|---|---|---:|---:|---|---:|
| Chat | Success | 31.6 s | 32.2 s | Four-character deterministic response | $0 provider charge; local model compute excluded |
| Research | Success | 0.06 s | 14.5 s | One official-source lookup; 207-character answer | $0 measured model charge; web-tool cost not exposed |
| Image | Success | Not recorded | About 38 s | `gpt-image-2-text-to-image`, 1K, persisted to private Gallery | $0.03 estimated |

Image data came from the completed live feature verification before R0 and was
not rerun to avoid unnecessary provider spend. Kie.ai did not return an actual
settled cost in the response; `$0.03` is Aio's configured estimate.

## Method

- Browser timings used headless Chromium against `http://localhost:3000/app`.
- Chat completion waited for the assistant message and streaming indicator to
  finish.
- Research completion waited for the progress card's `Research complete`
  state and a non-empty report.
- Docker startup was tested both without required hosted secrets and with safe
  non-secret placeholder configuration.
- Supabase verification reset an empty local database, applied every tracked
  migration in order, ran lint with `--fail-on warning`, then stopped local
  containers.

## Limits

- One sample per live workflow; no p50/p95 claim.
- Local hardware and warm caches affect latency.
- Research web-tool cost is unavailable in current telemetry.
- Local LM Studio compute/electricity is not converted into USD.
- Image cost is estimated, not reconciled provider billing.
- R3 must replace manual timing with per-run telemetry, cost attribution, and
  repeatable evaluation datasets.
