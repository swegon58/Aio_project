# Research Output Skeletons & Examples

## Type 1: Game Version Update Report

```markdown
### Version X.Y Release Summary

**Key Findings:**
- New characters added: [Character names]
- New game modes/features: [Feature list]
- Balance changes affecting existing content
- Bug fixes from previous patch notes
- Server stability improvements noted by players

**Pricing/Plans (if applicable):**
- Version is free-to-play with optional in-app purchases
- Battle pass available for $X USD equivalent
- No additional subscriptions required

**Notable Details:**
[Community sentiment, controversial changes, performance notes]

**Sources:**
1. [Official patch notes URL]
2. [Community discussion forum post]
3. [YouTube walkthrough or review]
```

### Example - Honkai: Star Rail v2.1 (Draft)

```markdown
### Version 2.1 Release Summary

**Key Findings:**
- New storyline chapters continue main plot
- Two new characters added to playable roster
- New enemy types in Penacony underground area
- Battle difficulty adjustment for higher tier content

**Pricing/Plans (if applicable):**
- Standard free update (no additional purchase required)
- Optional battle pass for version ($10 USD equivalent)

**Notable Details:**
[Community received new character design positively]
[Some players reported performance issues on mobile devices]

**Sources:**
1. [HoYoverse official blog post URL]
2. [Reddit r/HonkaiStarRail discussion thread]
3. [SteamDB version notes]
```

## Type 2: SaaS Platform Pricing Comparison

```markdown
### Product X vs Competitor Y Pricing Analysis

**Key Findings:**
- Product X: $X/mo for basic tier, free trial of X days
- Competitor Y: Free forever with limitations, paid tiers from $Y/mo
- Product Z: Usage-based pricing at $Z per conversation/task

**Pricing Model Analysis:**
[Per-seat vs per-task vs conversation-based comparison]
[Hidden costs and usage caps to watch for]

**Notable Details:**
[Unique features that justify premium pricing]
[Common complaints about hidden charges]

**Sources:**
1. [Official pricing page URL]
2. [Competitor A official pricing URL]
3. [Independent review site comparison]
```

### Example - Zapier vs Tidio (Draft)

```markdown
### Zapier vs Tidio Pricing Analysis

**Key Findings:**
- Zapier: Free tier forever with 100 tasks/mo, paid from $19.99/mo
- Tidio: Free trial 14 days, then $24/mo for conversational AI
- Crisp: Per-seat pricing ~$49-200+/mo depending on features

**Pricing Model Analysis:**
- Zapier: Task-based (automations counted as "tasks")
- Tidio: Conversational messages billed separately from webhooks
- Crisp: Per-seat model with add-on modules

**Notable Details:**
[Free forever plans often have hidden usage caps]
[Enterprise pricing requires direct contact for both platforms]

**Sources:**
1. [Zapier pricing page URL]
2. [Tidio pricing page URL]
3. [Crisp pricing page URL]
```

## Best Practices

### ALWAYS include:
- Direct links to primary source URLs (not just descriptions)
- Dates/version numbers for time-sensitive information
- Source credibility indicator (official/community/aggregator)

### NEVER include:
- Internal tool names or invocation details unless asked
- Model/backend architecture when user asks "what you're running on"
- Lengthy introductions to research findings

### When in doubt about accuracy:
- Cross-check with at least 2 independent sources
- Flag unconfirmed information clearly as such
- Prefer official documentation over aggregators
